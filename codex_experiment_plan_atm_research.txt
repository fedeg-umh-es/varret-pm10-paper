TASK 1 — ADD STL+RIDGE MODEL
════════════════════════════════════════════════════════════════════════════════

Create src/models/stl_ridge.py implementing a Seasonal-Trend decomposition +
Ridge direct model. This model is expected to populate the desirable zone
(skill > 0, α ≥ 0.8) because it explicitly separates the seasonal component
from the residual before fitting Ridge, preserving seasonal amplitude.

Algorithm for one rolling-origin fold:
  1. Decompose training series y_1,...,y_T with STL(period=7, robust=True)
     using statsmodels.tsa.seasonal.STL. Extract seasonal component S.
  2. Compute residual R = y_train - S.
  3. Build supervised dataset from R lags:
     features = [R[t], R[t-1], ..., R[t-6]] (7 lags)
     target   = R[t+h]   (residual at horizon h)
     Apply StandardScaler to features.
  4. Fit Ridge(alpha=1.0) on the residual supervised dataset.
  5. Prediction for horizon h:
     residual_pred = Ridge.predict(residual_lags_at_origin)
     seasonal_pred = y[origin - 7 + h]   (= seasonal naive on raw series)
     y_pred_h      = residual_pred + seasonal_pred

Add STLRidgeForecaster class with:
  __init__(self, season_length=7, ridge_alpha=1.0, n_lags=7)
  fit(train_df)          — takes the full training DataFrame with date,pm10
  predict_horizon(h)     — returns float y_pred for horizon h

Then modify scripts/01_generate_e1_rr_lags_only_predictions.py to include
stl_ridge as a fourth model inside _fit_predict_models(). The model name in
predictions.csv should be "stl_ridge_direct".

Note: STL requires at least 2×period = 14 observations; it is always satisfied
since MIN_TRAIN_ROWS = 365.

TASK 3 — ADD MAE SKILL
════════════════════════════════════════════════════════════════════════════════

In scripts/01_generate_e1_rr_lags_only_predictions.py, extend
_build_skill_summary() to also compute MAE-based skill alongside RMSE-based
skill. Add column "mae_skill" to the skill_summary output.

MAE skill:
  mae_model    = mean(|y_true - y_pred_model|)
  mae_baseline = mean(|y_true - y_pred_persistence|)
  mae_skill    = 1 - mae_model / mae_baseline

Rationale: MAE is less penalised by variance shrinkage than RMSE. If MAE_skill
≈ RMSE_skill for HGB/Ridge, it confirms the skill is not purely from
variance-flattening. If MAE_skill >> RMSE_skill, it shows the model gains
RMSE advantage primarily by reducing forecast variability.

Output: mae_skill column added to outputs/metrics/skill_summary.csv

TASK 5 — DIEBOLD-MARIANO SIGNIFICANCE TEST (fills GAP 3)
════════════════════════════════════════════════════════════════════════════════

Create scripts/05_dm_significance.py

For each (dataset, model, horizon) where model != persistence:
  Merge predictions for (model, horizon) with predictions for (persistence, horizon)
  on (fold, date) to get paired loss differences.

  d_t = (y_true - y_pred_persistence)^2 - (y_true - y_pred_model)^2

  DM statistic with Harvey-Leybourne-Newbold (1997) small-sample correction:
    n = number of paired observations
    d_bar = mean(d_t)
    gamma_0 = autocovariance of d_t at lag 0
    # For multi-step horizons h > 1, include MA(h-1) autocovariance correction:
    var_d = (1/n) * sum_{k=-(h-1)}^{h-1} gamma_k
    DM_stat = d_bar / sqrt(var_d)
    # HLN correction factor:
    hlncorrection = sqrt((n + 1 - 2*h + h*(h-1)/n) / n)
    DM_hlncorrected = DM_stat * hlncorrection
    p_value = 2 * (1 - t.cdf(abs(DM_hlncorrected), df=n-1))

  Apply Benjamini-Hochberg (1995) FDR correction across all tests within
  each dataset (alpha_bh = 0.05).

Output columns: dataset, model, horizon, n_pairs, dm_stat, dm_pval_raw,
                dm_pval_bh, dm_significant (bool, BH-adjusted p < 0.05)

Save to: outputs/tables/dm_significance_all_stations.csv

Expected finding: HGB/Ridge will show statistically significant positive skill
(dm_significant = True) for h ≥ 2 at most stations, confirming that the
positive RMSE skill is real — not just sampling noise — even as α collapses.

TASK 7 — MURPHY DECOMPOSITION (new experiment, levels up the paper)
════════════════════════════════════════════════════════════════════════════════

Create scripts/07_murphy_decomposition.py

Murphy (1988) decomposes MSE into three additive components:
  MSE = (ȳ_pred - ȳ_true)²  +  (σ_pred - ρ·σ_true)²  +  (1 - ρ²)·σ_true²
      = Bias²               +  Conditional Bias²      +  Irreducible error²

where ρ = Pearson correlation(y_pred, y_true), σ = std.

For each (dataset, model, horizon):
  Compute:
    bias_sq         = (mean_pred - mean_true)^2
    cond_bias_sq    = (std_pred - rho*std_true)^2
    irreducible_sq  = (1 - rho^2) * var_true
    mse_total       = bias_sq + cond_bias_sq + irreducible_sq

  Verify: abs(mse_total - mse_from_residuals) < 1e-6 (sanity check)

  Also compute: rho, std_pred, std_true, alpha = (std_pred/std_true)^2

Output: outputs/tables/murphy_decomposition_all_stations.csv
Columns: dataset, model, horizon, mse, bias_sq, cond_bias_sq, irreducible_sq,
         rho, std_pred_ug, std_true_ug, alpha

Key insight: For HGB/Ridge, cond_bias_sq will be large (predictions too flat
relative to truth). For STL+Ridge/SARIMA, cond_bias_sq should be smaller.

Create Figure: scripts/plot_murphy_decomposition.py
  Stacked bar chart: MSE decomposed into 3 components
  One panel per model, X-axis = horizon h, colors = {bias, cond_bias, irreducible}
  Save to: outputs/figures/figure_murphy_decomposition.pdf and .png

EXPECTED KEY FINDINGS (HYPOTHESES)
════════════════════════════════════════════════════════════════════════════════

After running Tasks 1-12, the paper should show:

H1  STL+Ridge achieves α > 0.5 at h=1 (and possibly α ≥ 0.8), confirming
    that variance collapse is NOT inevitable — it is a consequence of the
    lags-only paradigm, not of the ML approach per se.
    If H1 is TRUE: the paper argues "decompose before fitting to retain dynamics"
    If H1 is FALSE: "even with explicit seasonality, daily residuals collapse"
    — both are publishable findings, with the second being more negative.

H2  HGB/Ridge recall at P75 drops substantially vs persistence (expected <0.5
    at h=7 based on α≈0.10). STL+Ridge recall > HGB/Ridge recall.
    This provides the operational consequence currently missing from the paper.

H3  DM tests confirm HGB/Ridge skill is statistically significant for h ≥ 2,
    demonstrating the paradox explicitly: significantly better RMSE than
    persistence, yet operationally degraded episode detection.

H4  Collapse rates remain >90% for HGB/Ridge under any threshold from 0.3
    to 0.7, making the α=0.5 choice robust.

H5  Murphy decomposition shows that HGB/Ridge fail primarily through
    conditional bias (σ_pred << σ_true), not unconditional bias, linking
    the finding to established forecast verification theory.
